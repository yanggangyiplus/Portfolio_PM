### Phase 2: 기술 검토 및 아키텍처 설계 (2024.04.06 ~ 2024.05.20)

#### 기술 스택 검토 및 선택 근거

**프론트엔드: React + TypeScript (검토)**
- 선택 근거: 컴포넌트 재사용성, 타입 안정성
- 대안 검토: Vue.js (학습 곡선 고려), Angular (복잡도 높음)
- 결정 이유: 팀 내 React 경험자 다수, 생태계 풍부

**백엔드: Node.js + Express (검토)**
- 선택 근거: JavaScript 통일, 빠른 개발 속도
- 대안 검토: Python Django (학습 필요), Java Spring (과도한 복잡도)
- 결정 이유: 프론트엔드 개발자와 기술 스택 공유 가능

**데이터베이스: PostgreSQL (검토)**
- 선택 근거: 관계형 데이터 구조, 트랜잭션 지원
- 대안 검토: MongoDB (NoSQL, 관계형 데이터 필요), MySQL (기능 제한)
- 결정 이유: 사용자-게시글-댓글 관계 모델링 필요

#### 아키텍처 설계

**시스템 아키텍처 다이어그램 작성**
- 전체 시스템 구조 설계
- 컴포넌트 간 상호작용 정의
- 데이터 흐름 설계

**인프라 아키텍처 검토**
- AWS 서비스 검토 (EC2, RDS, S3, CloudFront)
- 배포 전략 검토
- 확장성 고려사항 정리
