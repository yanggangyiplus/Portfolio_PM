### Phase 2: 개발 (2024.04.06 ~ 2024.05.20)

#### 기술 스택 선택 및 근거

**프론트엔드: React + TypeScript**
- 선택 근거: 컴포넌트 재사용성, 타입 안정성
- 대안 검토: Vue.js (학습 곡선 고려), Angular (복잡도 높음)
- 결정 이유: 팀 내 React 경험자 다수, 생태계 풍부

**백엔드: Node.js + Express**
- 선택 근거: JavaScript 통일, 빠른 개발 속도
- 대안 검토: Python Django (학습 필요), Java Spring (과도한 복잡도)
- 결정 이유: 프론트엔드 개발자와 기술 스택 공유 가능

**데이터베이스: PostgreSQL**
- 선택 근거: 관계형 데이터 구조, 트랜잭션 지원
- 대안 검토: MongoDB (NoSQL, 관계형 데이터 필요), MySQL (기능 제한)
- 결정 이유: 사용자-게시글-댓글 관계 모델링 필요

#### 개발 진행 상황

**Week 1-2: 인프라 구축**
- 개발 환경 설정
- 데이터베이스 스키마 구현
- CI/CD 파이프라인 구축 (GitHub Actions)

**Week 3-4: 인증 시스템 개발**
- 카카오 OAuth 2.0 연동 완료
- 네이버 OAuth 2.0 연동 완료
- 구글 OAuth 2.0 연동 완료
- JWT 토큰 기반 인증 구현

**Week 5-6: 게시글 기능 개발**
- 게시글 CRUD API 구현
- 이미지 업로드 기능 (AWS S3 연동)
- 카테고리 관리 기능

**Week 7: 댓글 기능 개발**
- 댓글 CRUD API 구현
- 답글 기능 구현
- 좋아요 기능 구현

#### 개발 중 이슈 및 해결

**이슈 1: OAuth 콜백 처리 지연**
- 문제: 카카오 OAuth 콜백 시 간헐적 타임아웃 발생
- 원인 분석: 세션 저장소 Redis 미사용으로 인한 메모리 부족
- 해결: Redis 도입 및 세션 관리 최적화
- 결과: 응답 시간 3초 → 0.5초로 개선

**이슈 2: 이미지 업로드 성능**
- 문제: 다중 이미지 업로드 시 서버 부하 증가
- 원인 분석: 동기식 업로드 처리
- 해결: 비동기 업로드 및 이미지 압축 최적화
- 결과: 업로드 시간 50% 단축

### Phase 3: 테스트 및 배포 (2024.05.21 ~ 2024.06.15)

#### 테스트 계획 및 실행

**단위 테스트**
- 테스트 커버리지: 75%
- 주요 테스트 항목: API 엔드포인트, 비즈니스 로직

**통합 테스트**
- OAuth 플로우 전체 테스트
- 게시글 작성 → 댓글 작성 플로우 테스트

**사용자 테스트**
- 베타 테스터 30명 모집
- 테스트 기간: 2주
- 피드백 수집: 설문조사 + 인터뷰

#### 배포 과정

**인프라 구성**
- AWS EC2 (t3.medium) - 애플리케이션 서버
- AWS RDS (PostgreSQL) - 데이터베이스
- AWS S3 - 이미지 저장소
- CloudFront - CDN

**배포 파이프라인**
- GitHub Actions를 통한 자동 배포
- Blue-Green 배포 전략 적용
- 롤백 계획 수립

**배포 일정**
- 2024.06.10: 스테이징 환경 배포
- 2024.06.12: 프로덕션 배포 (소수 사용자)
- 2024.06.15: 전체 오픈
