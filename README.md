## 기획 역량 요약

### 서비스 기획
- **요구사항 분석**: 사용자 니즈 파악 및 비즈니스 요구사항 정의
- **기능명세서 작성**: FUNC-001부터 체계적으로 기능을 명세하여 작성
- **IA 설계**: 정보구조도 설계 및 ERD 작성
- **UX 플로우 설계**: 핵심 사용자 플로우 정의 및 예외 처리 설계

### UX 설계
- **와이어프레임 제작**: Figma를 활용한 와이어프레임 제작
- **사용자 플로우**: 핵심 플로우 다이어그램 작성
- **디자인 시스템**: 색상 팔레트, 폰트, 컴포넌트 가이드라인 정의
- **반응형 디자인**: 모바일/태블릿/데스크톱 최적화 설계

### 사용자 연구
- **사용자 행동 분석**: 데이터 기반 사용자 행동 패턴 분석
- **Pain Point 도출**: 사용자 불편사항 및 개선점 도출
- **개선안 제시**: Before → After 개선안 설계 및 문서화
- **사용자 테스트**: 프로토타입 테스트 및 피드백 반영

### 문서화
- **API 명세서**: RESTful API 엔드포인트 상세 명세
- **기능명세서**: 각 기능별 상세 명세서 작성
- **IA 다이어그램**: 정보구조도 및 ERD 다이어그램
- **UX 플로우 다이어그램**: 사용자 플로우 시각화

## 포트폴리오 구조

각 프로젝트는 다음 구조로 구성되어 있습니다:

```
프로젝트명/
├── README.md              # 프로젝트 개요 및 설명
├── IA/                     # 정보 구조도
├── UX_Flow/                # 사용자 플로우
├── Wireframes/             # 와이어프레임 (또는 Screens)
├── Requirements/           # 기능명세서
├── Survey_Research/        # 사용자 조사 (또는 Market_Analysis)
├── Roadmap/                # 로드맵 (또는 KPI_Definition)
└── Presentation/           # 발표 자료
```
